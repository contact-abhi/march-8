API ENDPOINTS
================================================================================

GET /api/reports
  • Fetches all traffic reports
  • Returns array of report objects
  • Auto-refreshed every 5 seconds
  • Fallback to sample data if DB unavailable

POST /api/reports
  • Creates new traffic report
  • Validates location and traffic level
  • Returns created report with ID
  • Error handling with user feedback

================================================================================
DATABASE SCHEMA
================================================================================

traffic_reports table:
  • id (UUID) - Primary key
  • latitude (DECIMAL) - Location
  • longitude (DECIMAL) - Location
  • traffic_level (VARCHAR) - 'low', 'medium', 'heavy'
  • description (TEXT) - Optional notes
  • timestamp (TIMESTAMPTZ) - When reported
  • created_at (TIMESTAMPTZ) - Creation time

Indexes:
  • timestamp DESC - For recent reports
  • location (lat, lng) - For geographic queries
  • traffic_level - For filtering
  • date/hour composite - For statistics

Row Level Security:
  • Public read access (select)
  • Public insert access (create)
  • Public delete access (delete)
  • Can be restricted in production
